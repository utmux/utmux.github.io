C++中的名称空间

一般来说，名称空间服从最小化访问原则，对于名称空间有以下三种的访问方法：

  1. 将 using namespace xx 放在该函数定义之前，相当于在全局定义了该文件中的函数都默认使用 xx 的名称空间，当时一般不这么做。
  2. 在特定的函数内部开头处使用 using namespace xx，表示该函数使用该名称空间。当时更加好的方法时只导入该名称空间中用到的一些变量，比如：using namespace std::cin 这种只导入 cin 而不是导入整个名称空间的。
  3. 使用名称空间前缀来访问名称空间的元素，例如 std::cin 或者 std::endl 来访问该名称空间中定义的对象和类。

C++中整数类型的选择
  1. 通常，int 被设置为对目标计算机而言最为“自然”的长度。自然长度（natural size）指的是计算机处理起来效率最高的长度。如果没有非常有说服力的理由来选择其他类型，则应使用 int。
  2. 如果变量表示的值不可能为负，如文档中的字数，则可以使用无符号类型，这样变量可以表示更大的值。万万不能自己为了节省空间说改用 unsigned 这个是非常容易出错的行为。
  3. 如果知道变量可能表示的整数值大于 16 位整数的最大可能值，则使用 long。即使系统上 int 为 32 位，也应这样做。这样，将程序移植到 16 位系统时，就不会突然无法正常工作。（根据 ISO 的标准 long 必须大于 32 位）
  4. 如果要存储的值超过 20 亿，可使用 long long。
  5. 通常，仅当有大型整型数组时，才有必要使用 short。（数组是一种数据结构，在内存中连续存储同类型的多个值。）如果节省内存很重要，则使用 short 而不是使用 int，即使它们的长度是一样的。
