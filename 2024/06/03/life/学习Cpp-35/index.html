String 类

C++在 C 语言的头文件 string.h（也就是 C++中的 cstring）中添加了大量的方法，我们这里来介绍一下这些方法。

16.1.1 字符串构造方法

我们首先来看一下字符串类的构造函数，string 类有 7 个构造函数（string是basic_string<char>的一个具体化模板），string 类分别重载了 [] 和 =，+= 以及 == 运算符，分别如下：

  1. string(const char *s)：根据 s 指向的字符串（以\0 结尾）创建 string 类

  2. string(size_type n, char c)：创建一个包含 n 个元素的字符串，每个字符都为 c

  3. string(const string&str)：复制构造函数

  4. string()：默认构造函数，创建一个空字符串，长度为 0

  5. string(const char *s, size_type n)：初始化一个 string，字符串为 s 的前 n 个元素（如果 n 超过了\0 仍然会继续读取）

  6. 使用迭代器初始化

  7. string(const string &str, string size_type pos = 0, size_type_n = npos)：将一个 string 对象初始化为对象 str 中从位置 pos 开始到结尾的字符，或从位置 pos 开始的 n 个字符

  8. string(string && str) noexcept ：C++11 新增的，这种构造函数被称为 移动构造函数，将一个 string 对象初始化 str，区别之前那个 string 就是使用右值初始化，它不保证将 str 视为 const，且确保不会抛出异常，主要应用场景是创建一个临时的 string 变量，并能够利用 noexcept 提高效率。

  9. 列表初始化

值得注意的是 string 对象的最大长度，其由 string::nops 指定，通常是最大的 unsigned int 值。string 版本的 getline 会从输入中读取字符，并存储到目标 string 中，知道下面三种情况：

  1. 到达文件为即 EOF，此时 istream 的 eofbit 将会被设置，此时 fail()和 eof()方法都会返回 true
  2. 遇到分解字符（默认为\n），此时会删除\n，且会从输入缓冲区丢弃该字符
  3. 读取的字符字数达到 string::nops 定义的最大允许值，此时会设置 istream 的 failbit，fail()返回将会返回 true

istream 类中有用于标记符来标记读取中遇到的情况，如果读取到文件末尾将设置 eofbit，如果检测到输入错误，将设置 failbit（比如输入的类型不匹配），如果出现无法识别的错误（比如硬盘故障），此时会设置 badbit，一切顺利才会设置 goodbit。

string 版本的 operator>>() 和字符数组版本的类似，不断读取，遇到空白字符就停止读取，并将空白字符留在缓冲区中。

16.1.3 使用字符串

string 对象不但重载了上面说到的一些运算符，还重载了比较运算符，比如：<, > 等等。string 类中确定长度可以使用方法 length()或者 size()。string 中查找内容，可以使用 find()方法，其中 find 方法有多个重载的版本，如下所示：

  1. size_type find(const string &str, size_type_pos = 0) const：从 pos 的位置开始查找字符串 str，如果找到返回首字母的索引，否则返回 string::nops
  2. size_type find(const char *s, size_type pos = 0)：类似上面
  3. size_type find(const char *s, size_type pos = 0, size_type n)：从 pos 开始，只查找 str 的前 n 个字符，返回和上面一样
  4. size_type find(char ch, size_type pos = 0) const：查找字符 ch，如果找到返回索引，否则返回 string::nops

除了 find 方法，还提供了 rfind()，find_first_of()，find_last_of()，find_first_not_of() 和 find_last_not_of() 方法，rfind() 类似于 find 不过是查找字符串最后一次出现的位置，find_first_of() 方法是查找参数中任何一个字符首次出现的位置，比如 find_first_of("01234567890")代表查找其中是否有数字。find_last_of()与之相反。find_first_not_of()返回第一个不在参数中的字符。

16.1.4 string 的其他功能

除了上面这些功能，string 有下面这些功能（常用的文本增删查改都有）：

  • 删除字符串的部分或者全部内容
  • 用一 个字符串的部分或全部内容替换另一个字符串的部分或全部内容
  • 将数据插入到字符串中或删除字符串中的数据
  • 将一个字符串的部分或全部内容与另一个字符串的部分或全部内容进行比较
  • 从字符串中提取子字符串
  • 将一个字符串中的内容复制到另一个字符串中
  • 交换两个字符串的内容

当字符串增加内容时，会遇上类似于 realloc 的问题，如果邻近的内存不够，该此时就需要重新分配内存，为了防止执行大量的操作，效率将会大大降低，因此实际运行中 C++给 string 分配的是一个比 string 中字符串大的内存块。然而如果字符串不断增大，超出这个内存块，此时程序将会分配一个新的大小为原来 两倍 的内存块。string 中含有两个方法：capacity()返回当前分配给字符串内存块的大小。reserve()方法用来保证请求的内存块的大小必须大于等于该长度，比如 reserve(50)执行后，该 string 变量的在我电脑上的占用就是 50。

string 类还有一个非常重要的方法，c_str() 方法，返回一个指向该 string 字符串的 C 风格字符串指针。这个方法经常被使用，比如将 string 转化为 C 类型的字符串。

16.2.2 智能指针的注意事项

为什么有这么多种智能指针呢？（实际上有四种，除了上面提到的三种，还有一种是：weak_ptr），以及为什么 auto_ptr 会被抛弃呢，答案就是如果有两个智能指针指向同一个地址此时会尝试去释放两次内存：

此时我们几种方法来解决这个问题（这就是为什么有这么多类型的智能指针）：

  1. 重载赋值运算符，使之进行深复制，这样两个智能指针各自指向不同的对象，释放的时候就不会释放同一个地址呢。
  2. 建立所有权(ownership)的概念，对于一个对象，只能有一个智能指针可以拥有它。赋值操作会转让所有权，只有拥有所有权的智能指针能够 访问和释放 该对象。这就是用于 auto_ptr 和 unique_ptr 的策略。
  3. 创建智能更高的指针，跟踪应用该对象的智能指针数。这称为引用计数（reference counting），仅再引用技术为 1 时才会调用 delete 去释放内存。这就是 shared_ptr 采用的策略。

当然复制构造函数采取的策略和上面的赋值运算符是一样的。
