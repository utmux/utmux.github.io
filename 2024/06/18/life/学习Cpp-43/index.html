Tips: 注意写入类到文件中

如果我们使用 write，将某个类所在的地址写入到文件中，分为两种情况，一种是该类没有虚函数，此时该类只有数据成员会被写入，而方法不会被写入。另一种是该类存在虚函数，此时也将复制虚函数指针表。当下一次运行，虚函数所处的内存位置发生变化时，此时从文件中读取这个类来访问虚函数时就会发生错误。

随机存取

为使编程更具创造性，该示例将以读写模式打开文件，以便能够读取和修改记录。为此，可 以创建一个 fstream 对象。fstream 类是从 iostream 类派生而来的，而后者基于 istream 和 ostream 两个类， 因此它继承了它们的方法。 它还继承了两个缓冲区，一个用于输入，一个用于输出，并能同步化这两个缓 冲区的处理。也就是说，当程序读写文件时，它将协调地移动输入缓冲区中的输入指针和输出缓冲区中的 输出指针。

Tips：C 和 C++在修改文件指针区别

在 C 语言中，使用 fseek 修改文件指针

参数说明：

  • stream：指向 FILE 结构的指针，表示要操作的文件流。
  • offset：偏移量，表示从 whence 指定的位置开始跳转的字节数。
  • whence：指定偏移量 offset 的起始位置，可以是以下值之一：
    1. SEEK_SET：文件开头。
    2. SEEK_CUR：当前文件位置。
    3. SEEK_END：文件末尾。

可以看出 offset 偏移量和基址 whence 都是 long 或者 int，而 C++中为了便于移植，修改了这个限制，原因是对于有些文件系统，整数参数无法提供足够的信息。因此 streamoff 和 streampos 允许是结构或类类型，条件是它们允许一些基本的操作，如使用整数值作为初始值等。随后，老版本的 istream 类被 basic_istream 模板取代，streampos 和 streamoff 被 basic_istream 模板取 代。然而，streampos 和 streamoff 继续存在，作为 pos_type 和 off_type 的 char 的具体化（basic_istream 模板类定义了 pos_type 和 off_type 类型，用来表示文件位置和文件偏移）。同样，如果将 seekg( )用于 wistream 对象，可以使用 wstreampos 和 wstreamoff 类型。
