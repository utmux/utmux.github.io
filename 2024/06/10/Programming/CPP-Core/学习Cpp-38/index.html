    Tips: rbegin()和 rend()

    rbegin()返回的是指向超尾的反向迭代器，rend()返回的是指向第一个元素的反向迭代器。值得注意的是 rbegin()和 end()返回的是同样的值，同样 rend()和 begin()返回的值也是一样，但是不同的是类型，前者返回的是一个 reverse_iterator，而后者返回 iterator。

    对于反向迭代器，我们需要有一个小修改，这是因为 rbegin()指向的是最后一个元素的超尾，该地址是不应该被访问的，为了解决这个问题，reverse_iterator 会默认将 *p++（先访问在加）替换成 *--p（先递减再访问）

  2. insert_iterator
    insert_iterator 将元素插入到 insert_iterator 构造函数的 参数指定的位置前面。提示：可以用 insert_iterator 将复制数据的算法转换为插入数据的算法。

  3. back_insert_iterator

    back_insert_iterator 将元素插入到容器尾部，back_insert_iterator 只能用于 允许在尾部快速插入的容器（快速插入指的是一个时间固定的算法）。该迭代器将会默认容器存在 push_back()方法

  4. front_insert_iterator
    front_insert_iterator 将元素插入到容器的前端。front_insert_iterator 只能用于允许在起始位置做时间固定插入的容器类型，vector 类不能满足这种要求，但 queue 满足。该迭代器将会默认容器存在 push_front()方法

容器种类

STL 具有容器概念和容器类型。不能将任何类型的对象存储在容器中，具体地说，类型必须是可复制构造的和可赋值的。基本类型满足这些要求；只要类定义没有将复制构造函数和赋值运算符声明为私有或保护的，则也满足这种要求。C++11 改进了这些概念，添加了术语可复制插入（CopyInsertable）和可移动插入（MoveInsertable），但这里只进行简单的概述。

基本容器不能保证其元素都按特定的顺序存储，也不能保证元素的顺序不变，但对概念进行改进后，则可以增加这样的保证。所有的容器都提供某些特征和操作。下表对容器一些 通用特征 进行了总结。其中，X 表示容器类型，如 vector；T 表示存储在容器中的对象类型；a 和 b 表示类型为 X 的值；r 表示类型为 X& 的值；u 表示类型为 X 的标识符（即如果 X 表示 vector<int>，则 u 是一个 vector<int> 对象）。

Tips: 编译时间，固定时间和线性时间

如果复杂度为编译时间，则操作将在编译时执行，执行时间为 0。固定复杂度意味着操作发生在运行阶段，但独立于对象中的元素数目。线性复杂度意味着时间与元素数目成正比。即如果 a 和 b 都是容器，则 a = = b 具有线性复杂度，因为 = = 操作必须用于容器中的每个元素。实际上，这是最糟糕的情况。如果两个容器的长度不同，则不需要作任何的单独比较。

C++11 在上表上新增的通用容器要求。在这个表中，rv 表示类型为 X 的非常量右值，如函数的返回值。另外，在表中，要求 X:: iterator 满足正向迭代器的要求，而以前只要求它不是输出迭代器。

tips：复制构造函数和移动构造函数

复制构造和复制赋值以及移动构造和移动赋值之间的差别在于，复制操作保留源对象，而移动操作可修改源对象，还可能转让所有权，而不做任何复制。如果源对象是临时的，移动操作的效率将高于常规复制。

序列

如果容器要求数据必须按照某种顺序排列（比如队列必须从队头出队尾进），此时就需要序列约束（sequence）。序列还要求其元素按严格的线性顺序排列，而分支结构的数据不是线性的。因为序列中的元素具有确定的顺序，因此可以执行诸如将值插入到特定位置、删除特定区间等操作。下面是序列的必须要求和可选要求：

容器介绍

  1. vector
    vector 是可反转容器，存在两个方法 rbegin()，rend()，除非其他类型有特殊优点能够更好满足程序要求，否则应该默认使用 vector。

  2. deque
    双端队列，支持随机访问，与 vector 不同的是，vector 中插入和删除是线性时间的，而 deque 对双端的开始位置的插入和删除是固定时间的。

  3. list
    线性序列，也是可反转容器，与 vector 不同的是，list 不支持数组表示法和随机访问（因为 list 底层结构是双向链表），每次插入和删除都需要将后面的数据进行搬移，从容器中插入和删除数据，迭代器指向元素将不变（双向链表的特性而已）。下面是 list 的一些成员函数：

    sort()，merge()，unique()，remove()还能 传入一个函数，来判断某个值是否应该被执行操作，传入的函数应该是谓词函数（返回值为 bool 的一元函数)

  4. forward_list
    单项链表，forward_list 不可反转，相比双向列表 list 更加简单，功能也更少。

  5. queue
    单向对内，默认底层是 dequeue（双向队列），可以更改（C++20 新增）。相比 dequeue，不允许随机访问队列元素，甚至不允许遍历队列，操作只限制在最基本的队列操作上：队尾添加元素，队首删除元素，查看队尾，查看队首，检查元素数目，测试队列是否为空。

  6. priority_queue
    支持操作和上面的 queue 一样，只不过会保持最大的元素总是在队首，和 queue 的区别是底层是 vector，而 queue 的底层为 dequeue。我们可以修改哪个元素放到队首的比较方式，如下：

  7. stack
    和 queue 类似，在头文件 stack 中声明，也是一个适配类 adapter，stack 只提供典型的栈接口，不允许随机访问，不允许遍历栈，只有入栈，出栈，查看栈顶的值，检查元素数目和检查栈是否为空。

  8. array(C++)
    array 在头文件 array 中定义，并非 STL 容器，因为其长度是固定的，因此 array 没有定义调整容器大小的操作（如 insert 和 push_back），但是定义了成员函数 operator [] 和 at()，并且适配了 STL 算法，使得 很多 STL 算法可以用于 array 对象（copy()和 foreach()等）
