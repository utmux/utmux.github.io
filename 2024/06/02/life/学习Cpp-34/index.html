类型转换运算符

C 语言中的类型转换运算符太过松散，没有限制，没有规范。C++中的类型转换相比更加严格有规范，C++中有四个类型转换运算符：

  • dynamic_cast
  • const_cast
  • static_cast
  • reinterpret_cast

下面来说一下上面四种转换运算符，首先是 dynamic_cast 这个的主要应用场景在之前就说过了。然后是 const_cast 运算符，该运算符仅有一种用途，改变值为 const 或者 volatile（相当于一个锁，当你想修改某个 const 时，转化一下，修改完再转换回去），语法和 dynamic_cast 语法一样，例如：

static_cast 运算符和上面两个的语法一样，能够进行一些强制类型转换，其强制类型转换的 前提是一方能够隐式转换到另一方（弱连通？），比如 int 能转化为 double，派生类能够转化到基类，枚举能够转换到 int。此时 static_cast 就能强制，将本来不可逆的转化强制转换回去。

reinterpret_cast 适用于天生危险的转化，该转换运算符不允许删除 const。例如访问某个地址的寄存器，其适合将 long 值转为为某种数据结构的指针。也可以将指针转化为某个整数，但是其有限制：

  1. 指针转化为整形或者浮点，该整形或者浮点必须能够容纳下该值，比如 32 位的指针，就必须使用 32 位以上的整形数据表示。

  2. 不能将函数指针转化为数据指针或者将数据指针转化位航速指针
