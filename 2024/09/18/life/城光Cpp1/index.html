    const 和 static 相关

    • const 变量可以在构造函数中初始化，但不能在定义时初始化。
    • const 变量可以用 static 变量代替，或者结合 static 使用，此时视为 static 类型。
    • static 变量必须在类外初始化。

    对于静态成员函数和静态数据成员，可以使用 . 操作符访问，但更推荐使用作用域解析运算符 :: 来访问。

    不宜使用内联的情况

    1. 如果函数体内的代码比较长，内联将导致内存消耗代价较高。
    2. 如果函数体内出现循环，执行函数体内代码的时间会比函数调用的开销大。

    inline 相关

    2. 虚函数（virtual）可以是内联函数（inline）吗？

    • 虚函数可以是内联函数，但当虚函数表现多态性时不能内联。
    • 内联是在编译期建议编译器进行内联，而虚函数的多态性发生在运行期，编译器无法预知运行期调用哪个代码，因此虚函数表现为多态性时（运行期）不可以内联。
    • inline virtual 唯一可以内联的情况是：编译器知道所调用的对象是哪个类（如 Base::who()），这只有在编译器具有实际对象而不是对象的指针或引用时才会发生。

    sizeof 相关

    • 在一个类中，虚函数本身、成员函数（包括静态与非静态）和静态数据成员都不占用类对象的存储空间。
    • 对于包含虚函数的类，不管有多少个虚函数，都只有一个虚指针（vptr），其大小为 4 字节（32 位操作系统）或 8 字节（64 位操作系统）。
    • 普通继承时，派生类继承了所有基类的函数与成员，需要按照字节对齐来计算大小。
    • 虚函数继承时，不管是单继承还是多继承，都继承了基类的 vptr。
    • 虚继承时，继承基类的 vptr。

    2. 友元函数（Friend Functions）

    友元函数是能够访问类的私有和保护成员的非成员函数。它们在类的声明中通过 friend 关键字定义。

    3. 非约束/约束友元函数

    3.1. 非约束友元函数

    非约束友元函数不依赖模板参数，它们独立于模板类。即使友元函数声明在模板类中，它也不使用模板参数。

    3.2. 约束友元函数

    约束友元函数使用模板参数，需要通过模板定义，才能访问模板类的成员。

    4. 非约束/约束模板友元函数

    4.1. 非约束模板友元函数

    模板类的友元函数可以是非约束的，即友元函数本身是模板，而它与模板类没有直接关联。

    4.2. 约束模板友元函数

    约束模板友元函数是指友元函数不仅是模板，而且依赖于模板类的参数。

    5. 友元类（Friend Class）

    友元类是允许另一个类访问本类的私有和保护成员的类。友元类可以访问所有成员函数和变量。

    6. 前向声明友元类中的某个方法

    可以通过前向声明，只让友元类中的某个方法访问目标类的私有或保护成员，而不是让整个友元类访问。

    Tips: 嵌套类的访问域和权限

    1. 访问控制
      嵌套类定义的位置（public，private 和 protected）决定了该嵌套类的作用域。而类中成员的定义决定了程序对成员的访问特性。
    2. 作用域
      如果嵌套类实在类中的私有部分声明的，则只有该类能够访问该嵌套类的成员，派生与其他类都无法访问。而如果实在 protected 中声明的，此时派生类也能看到。如果是 public，此时公共部分就能访问该嵌套类。
