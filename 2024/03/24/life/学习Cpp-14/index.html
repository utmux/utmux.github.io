函数的链接性

对于使用 static 修饰的函数，如果本文件中有对该函数的调用，其优先权是要高于其他文件的同名函数的。对于内联函数则不同，内联函数的本质是各个调用的地方都复制一份，可以直接在头文件中定义。

C++在根据函数名寻找函数时，如果用户定义了一个与库函数中同名的函数，C++将使用用户定义的函数，而不是库函数。这种情况叫做函数覆盖。

C 与 C++库的不同

对于 C 语言，编译器在寻找库的时候由于 C 中没有重载机制，这保证了 C 中所有的函数名都是唯一的。因此 C 的库函数符号一般为 _function，而对于 C++由于有重载存在，故其库中的函数符号包括了参数例如对于函数 spiff(int)和 spiff(double, double)，其函数名分别为：_spiff_i 和 _spiff_d_d，故如果 C++想要链接到 C 语言的库，必须这样写：

名称空间

名称空间是全局的，可以位于另一个名称空间中，在默认情况下其引用时外部的，除非它 引用了常量。

使用 using 声明引用名称空间会覆盖当前同名的函数或者变量定义等。

名称空间的使用原则

名称空间相当于把全局的变量和函数进行了分类，下面是一些指导

  • 使用在已命名的名称空间中声明的变量，而不是使用外部全局变量。（等于就是将全局变量放在名称空间中）
  • 使用在已命名的名称空间中声明的变量，而不是使用静态全局变量。（静态变量也是放在名称空间中）
  • 如果开发了一个函数库或类库，将其放在一个名称空间中。事实上，C++当前提倡将标准函数库 放在名称空间 std 中，这种做法扩展到了来自 C 语言中的函数。例如，头文件 math.h 是与 C 语言 兼容的，没有使用名称空间，但 C++头文件 cmath 应将各种数学库函数放在名称空间 std 中。实 际上，并非所有的编译器都完成了这种过渡。
  • 仅将编译指令 using 作为一种将旧代码转换为使用名称空间的权宜之计。 （也就是说以后调用函数常量要用完整名称空间）
  • 不要在头文件中使用 using 编译指令。 首先，这样做掩盖了要让哪些名称可用；另外，包含头文件的顺序可能影响程序的行为。如果非要使用编译指令 using，应将其放在所有预处理器编译指令 #include 之后。
  • 导入名称时，首选使用作用域解析运算符或 using 声明的方法。（即使用 :: 或者 using 导入具体使用的包） 对于 using 声明，首选将其作用域设置为局部而不是全局。
