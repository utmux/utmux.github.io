内联函数

内联函数节省了堆栈和跳转的开销，但是值得注意的是内敛函数会导致编译出的可执行文件以及运行时所占的内存大小变大，原因是如果有 10 个地方调用该内敛函数，则该内敛函数也会复制十次到调用位置，而不是跳转到某个地方执行。
应该有选择的使用内敛函数，内联函数时间太长节省的时间相对执行时间来说微不足道，太短则即使节省相对大部分时间，但是由于这个过程相当快，因此尽管节省了该过程的大部分时间，但节省的时间绝对值并不大，除非该函数经常使用。

引用变量参数的函数注意点

使用引用变量作为函数的参数时，值得注意的是，如果实参与引用参数不匹配，C++将生成临时变量。仅当函数的形参为 const 时才会，例如：

C++会在下面两种情况下生成临时变量：

  • 实参类型正确，但不能作为左值（左值参数是可被引用的数据对象，例如，变量、数组元素、结构成员、引用和解除引 用的指针都是左值。非左值包括字面常量（用引号括起的字符串除外，它们由其地址表示）和包含多项的 表达式。）这里 const 也属于左值，只是为不可修改的左值。
  • 实参的类型不匹配，但可以转化为正确的类型

这些临时变量只会在函数调用期间存在，此后便可以随意删除。

Tips：应尽可能使用 const

将引用参数声明为常量数据的引用的理由有三个：

  • 使用 const 可以避免无意中修改数据的编程错误；
  • 使用 const 使函数能够处理 const 和非 const 实参，否则将只能接受非 const 数据；
  • 使用 const 引用使函数能够正确生成并使用临时变量
