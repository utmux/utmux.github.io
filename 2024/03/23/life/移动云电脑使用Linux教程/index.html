移动云电脑使用 Linux 教程

引言

最近在逛咸鱼的时候发现了移动云电脑这个东西，价格是出奇的便宜，我看了下咸鱼上那些卖的基本上是管理员在卖，具有直接收回的权力，虽然我想直接冲一年，但是看到了评论区中说被商家污蔑挖矿结果账号被收回的还是让人担心。所以就只开了一个月。

我买云电脑的主要动机是学习 Linux，因为本机的内存太小，所以想买个云主机学习 Linux，结果云主机实在太贵，然后又在咸鱼上看到了这个，所以就买了。在买之前我先买了个 2C4G 的一个月 6 元测试了下，发现了

  1. 可以阻止自动关机
  2. 可以开启 Hyper V
  3. 可以开启内部 NAT 来配置虚拟机网络
  4. 尝试启动了下 mint linux 结果貌似是因为内存不足，最后试了下 Ubuntu 16.04，是可以正常启动，并能够通过 todesk 远程连接的。

发现基本上能够正常使用后，我就买了 12 月一个月的 8C16G 80G 系统盘 + 500 数据盘的那个版本。如果够好用我就尝试续费。

第一步优化一下电脑配置

首先在果壳剥壳网站中下载了 Optimizer 和 Wisecare365 关闭了原本电脑中的各种没用的服务。我没有关闭自带云电脑软件的开机启动，因为怕关掉后无法通过云电脑的 APP 连接。

第二步开启 hyper V

这里参考微软官方的关于在 win10 系统上开启 hyperv 的教程

第三步配置

这里是折腾的点，我一开始根据网上的教程设置 hyperv 的网络发现根本不行，这里告诉你如何设置，首先找一下 hyper v，nat 这两个关键词，如下面的链接所示：
如何设置 NAT 网络
设置完后在 hyper V 中选择内部，选择这个网络就 OK 了

本以为能够在 8 核心那个套餐也能启动，后来发现是 AMD 的处理器，然后根据 hyper V 的要求 AMD 的虚拟化技术在 Windows server2022 中才能实现。那个 8 核心的 AMD 处理器根本不行，本身系统是 Windows server 2016，所以说完全没有。况且卖家说是有 50Mbps 的带宽，实际上只有下载能够在非高峰期达标。上传最高时 4Mbps 左右，而且在高峰期基本为 0。所以说如果要买 8 核心的最好先问好处理器是否为 intel，系统版本。如果是 amd 需要 Windows11 或者 Windows server2022 以上。如果是 intel 的在 Windows server2016 以上即可。
