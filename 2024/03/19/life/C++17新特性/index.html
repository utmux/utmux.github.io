C++17新特性

下面是我读到关于C++14中模板具体化实例的特性联想到了C++17的新特性，问chatgpt得到的回答如下，作为一个mark，未来没准会用到。

C++17 相比于 C++14 引入了许多新特性和改进，这些变化旨在提高语言的性能、功能和易用性。以下是一些主要的改变和新增特性：

  1. 结构化绑定（Structured Bindings）:
    C++17 引入了结构化绑定，允许从数组、元组或结构体中直接创建变量的引用或副本。这简化了从复合数据类型中提取数据的过程。

  2. if constexpr:
    这是一个编译时的 if 语句，允许在编译时根据条件包含或排除代码，从而提高编译后的程序性能。

  3. 内联变量（Inline Variables）:
    C++17 允许在头文件中定义内联变量，这意味着它们可以在多个翻译单元中重复定义而不会引起链接错误。

  4. *lambda 捕获 this (包括 this):
    C++17 允许 lambda 表达式捕获 this 指针，这使得 lambda 可以访问成员对象和成员函数。

  5. auto 关键字的改进:
    C++17 修改了 auto 关键字的推导规则，特别是对于直接列表初始化。现在，如果花括号中只有一个元素，auto 将推导为该元素的类型，而不是 std::initializer_list。

  6. std::filesystem:
    C++17 引入了一个新的标准库，提供了跨平台的文件系统操作接口。

  7. 并行算法库:
    C++17 扩展了标准库算法，允许它们并行执行，以利用多核处理器的性能。

  8. std::optional 和 std::variant:
    这两个模板类提供了更安全和灵活的方式来表示可选值和多种可能类型的值。

  9. std::any:
    这是一个可以存储任何类型的容器，类似于 C 语言中的 void*，但更加安全和方便。

  10. if 初始化语句:
    C++17 允许在 if 语句中声明局部变量，这些变量的作用域仅限于 if 语句块。

  11. std::shared_mutex 和 std::shared_lock:
    这些新的同步原语提供了读写锁的功能，适用于读多写少的场景，提高了并发性能。

  12. std::string_view:
    这是一个非拥有的字符串视图，允许访问字符串数据而不进行复制。

这些只是 C++17 相对于 C++14 引入的一部分新特性。C++17 的目标是提高代码的性能、安全性和表达能力，同时保持与旧代码的兼容性。
