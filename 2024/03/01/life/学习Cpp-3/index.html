C++整数常量表示

C++中和 C 类似，其字面值整数可以分别用 8 进制（以 0 开头，后面的值小于 8），10 进制和 16 进制（以 0x 开头）。使用 cout 显示时，常量是什么进制，输出就是什么进制。对于要求指定某进制输出 cout，可以通过控制符 dec，hex，oct 来实现不同进制的显示。比如：

在 C++中十进制整数常量默认存储为 int 类型和 C 语言中的标准是一致的，除非存不下此时会存储为 long 类型，即默认在下面 int、long 或 long long 三种选择中选择，除非存不下选择后一个。在常量后面通过添加后缀：u，l，ll（大小写都可以，u 可以和 l，ll 组合）来指定存储类型。
而对于 8 进制和 16 进制整数常量，由于一般不考虑正负，其一般用能够存储该数的最小类型来表示：int、unsigned int long、unsigned long、long long 或 unsigned long long。

在 C++中没有关于字符 char 是否有符号的规定，这个需要根据自己写的程序和 C++编译器的实现来规定。

C++中的其他字符类型

为了兼容 Unicode 这些国际标准在 C++中有宽字符类型 wchar_t 和 wcin，wcout 来处理该类型 wchar_t（宽字符类型）可以表示扩展字符集。wchar_t 类型是一种 整数类型，它有足够的空间，可以表示系统使用的最大扩展字符集。
除此之外，在 C++11 中增加了 char16_t 和 char32_t，为了不仅仅满足普通的 Unicode 字符，实现更加长的字符，采取了这两种类型，该两种类型都是无符号类型，分别用大写和小写的 u 来表示。C++11 使用前缀 u 表示 char16_t 字符常量和字符串常量，如 u‘C’和 u“be good”；并使用前缀 U 表示 char32_t 常量，如 U‘R’和 U“dirty rat”。类型 char16_t 与/u00F6 形式的通用字符名匹配，而类型 char32_t 与/U0000222B 形式的通用字符名匹 配。

批注：wchar_t 会根据编译的系统平台不同而不同，在 Windows 上是两个字节，也就是 int 的一半，一般是可以容下 utf-8 中的一字节和二字节字符（一些希腊符号），但是中文主要在三字节字符中，少部分在四字节字符中。也就是 如果要完整支持中文，需要使用四字节，也就是 char32_t 的。在 Python 中，就是使用 char32_t，但是为了效率，做了一点 trick。上面三者对应的字符串为 std::wstring, std::u16string, std::u32string
