数组名和数组第一个元素的区别
  1. 数组声明使用数组名来标记存储位置；
  2. 对数组名使用 sizeof 将得到 整个数组的长度（以字节为单位）；
  3. 正如第 4 章指出的，将地址运算符&用于数组名时，将返回整个 数组的地址，例如&cookies 将返回一个 32 字节内存块的地址（如果 int 长 4 字节）

相比 int* arr 来表示数组，int arr[] 表示更加清楚，使用数组下标是等效于指针访问的

Tips

为了保证数组和指针指向的数据被不小心修改，使用 const 修饰符是一个非常好的习惯。

尽量使用 const

尽量使用 const 指针有下面两条原因：

  • 避免无意修改数据
  • 能够接受 const 和非 const 的实参，否则如果不加 const 则只能接受非 const 值，例如 foo(int []); 将不能接受 const arr[4];
Tips

在 C 和指针书上看到的一些技巧：

  1. 指针的声明应该为 int b, 而不应该把星号与 int 临近，例如在声明 int a, b; 中就能清楚的知道 a 是指针，而 b 则为一个整数类型的数据。
  2. 如果数组时编程时人为指定的，通过计算的到，则一般不用检查，而对于那些数组数据是通过外界用户输入数据的则一定要检查数组下界是否溢出。
结构体与指针

对于较小的结构体应该直接按值进行传递，而对于较大的结构体建议使用指针进行传递，避免过多的内存开销。类似的直接用等于符号将一个新的结构体赋值也会消耗一定的内存开销。
