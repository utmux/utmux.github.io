线程

线程概念

线程在Linux上本质就是进程，又叫做Light Weight Process（LWP），即轻量化进程。Linux中查看某个进程下的线程LWP号，即LWP号，通过ps -Lf pid查看。

线程是调度的最小单位，每个线程都有自己的栈，栈中保存了线程的运行环境，包括寄存器、指令指针、堆栈指针等。

进程和线程的区别

  1. 进程和线程是操作系统资源分配的基本单位。进程是操作系统资源分配的基本单位，而线程是调度的最小单位。
  2. 进程的创建通过fork实现，即便使用写时复制技术，其代价也比较高，因为其父子进程不是共享内存的需要复制内存页表和文件描述符之内的。而对于线程，其父子进程是共享内存的，所以创建线程代价更小也更加快速。通常情况下，其速度是创建进程的10倍以上。

线程之间的共享资源和非共享资源如下表所示

共享资源

资源类别 具体内容
进程 ID 和父进程 ID 所有线程共享相同的进程 ID（PID）和父进程 ID（PPID）。
进程组 ID 和会话 ID 所有线程共享相同的进程组 ID（PGID）和会话 ID（SID）。
用户 ID 和用户组 ID 所有线程共享相同的用户 ID（UID）和用户组 ID（GID）。
文件描述符表 所有线程共享相同的文件描述符表，可以访问相同的打开文件、套接字等。
信号处置 所有线程共享相同的信号处理函数（signal handlers）。
文件系统的相关信息 所有线程共享相同的文件权限掩码（umask）和当前工作目录。
虚拟地址空间 所有线程共享相同的虚拟地址空间，包括堆、全局变量、代码段（.text）等。
环境变量 所有线程共享相同的环境变量。
内存映射 所有线程共享相同的内存映射（如 mmap 映射的文件或共享内存）。
定时器 所有线程共享相同的进程定时器。

非共享资源

资源类别 具体内容
线程 ID 每个线程有唯一的线程 ID（TID）。
信号掩码 每个线程可以设置独立的信号掩码，决定哪些信号会被阻塞。
线程特有数据 每个线程可以有自己的线程局部存储（Thread Local Storage, TLS）。
error 变量 每个线程有独立的 errno 变量，用于存储最近一次系统调用的错误代码。
实时调度策略和优先级 每个线程可以设置独立的调度策略（如 SCHED_FIFO、SCHED_RR）和优先级。
栈 每个线程有独立的栈，用于存储局部变量和函数调用链信息。
寄存器状态 每个线程有独立的寄存器状态（如程序计数器、栈指针等）。
浮点环境 每个线程可以设置独立的浮点环境（floating-point environment）。
