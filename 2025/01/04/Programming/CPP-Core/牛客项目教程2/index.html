线程

线程概念

线程在 Linux 上本质就是进程，又叫做 Light Weight Process（LWP），即轻量化进程。Linux 中查看某个进程下的线程 LWP 号，即 LWP 号，通过 ps -Lf pid 查看。

线程是调度的最小单位，每个线程都有自己的栈，栈中保存了线程的运行环境，包括寄存器、指令指针、堆栈指针等。

进程和线程的区别

  1. 进程和线程是操作系统资源分配的基本单位。进程是操作系统资源分配的基本单位，而线程是调度的最小单位。
  2. 进程的创建通过 fork 实现，即便使用写时复制技术，其代价也比较高，因为其父子进程不是共享内存的需要复制内存页表和文件描述符之内的。而对于线程，其父子进程是共享内存的，所以创建线程代价更小也更加快速。通常情况下，其速度是创建进程的 10 倍以上。

共享资源

资源类别 具体内容
进程 ID 和父进程 ID 所有线程共享相同的进程 ID（PID）和父进程 ID（PPID）。
进程组 ID 和会话 ID 所有线程共享相同的进程组 ID（PGID）和会话 ID（SID）。
用户 ID 和用户组 ID 所有线程共享相同的用户 ID（UID）和用户组 ID（GID）。
文件描述符表 所有线程共享相同的文件描述符表，可以访问相同的打开文件、套接字等。
信号处置 所有线程共享相同的信号处理函数（signal handlers）。
文件系统的相关信息 所有线程共享相同的文件权限掩码（umask）和当前工作目录。
虚拟地址空间 所有线程共享相同的虚拟地址空间，包括堆、全局变量、代码段（.text）等。
环境变量 所有线程共享相同的环境变量。
内存映射 所有线程共享相同的内存映射（如 mmap 映射的文件或共享内存）。
定时器 所有线程共享相同的进程定时器。

非共享资源

资源类别 具体内容
线程 ID 每个线程有唯一的线程 ID（TID）。
信号掩码 每个线程可以设置独立的信号掩码，决定哪些信号会被阻塞。
线程特有数据 每个线程可以有自己的线程局部存储（Thread Local Storage, TLS）。
error 变量 每个线程有独立的 errno 变量，用于存储最近一次系统调用的错误代码。
实时调度策略和优先级 每个线程可以设置独立的调度策略（如 SCHED_FIFO、SCHED_RR）和优先级。
栈 每个线程有独立的栈，用于存储局部变量和函数调用链信息。
寄存器状态 每个线程有独立的寄存器状态（如程序计数器、栈指针等）。
浮点环境 每个线程可以设置独立的浮点环境（floating-point environment）。

线程的退出

线程会在下面这几种情况下自动退出：

  • 线程函数执行完毕
  • 调用 pthread_exit 函数
  • 调用 exit 函数（此时会导致进程终止，从而所有线程也一起退出）
  • 调用 cancel 函数
  • 主线程退出，此时所有线程都会退出

进程的连接

和进程类似，子线程也需要进行资源回收，在前面的进程中，子进程被父进程或者 init 父进程回收，而子线程则需要手动回收。其中父进程可以通过调用 wait 或者 waitpid 函数进行回收。而对于线程而言，其可以被父线程回收，也可以被其他子线程回收。

和进程类似，如果不回收子线程，也会产生僵尸线程。通过 pthread_join 函数可以回收子线程，并且可以获取子线程的返回值。

线程的分离

线程的分离函数为 pthread_detach()，参数为线程 ID，返回值为 0 表示成功，其他表示失败。
和 pthread_join 类似，其会标记传入的线程，并在线程结束后自动释放资源，不需要用户自己释放。当线程分离后，此时不能通过 pthread_join 来等待线程结束，否则会返回错误。

线程的取消

线程的取消和上面的分离差不多，只是通知线程取消，其会被标记，而不是保证立刻取消，该函数是非阻塞的，代表其只会返回这个线程是否被标记成功。

根据 man 手册的手法，一个线程是否可以取消，取决于它的两个属性，cancelablity 和 cancelstate。

  1. cancelablity
    对于该属性，其通过 pthread_setcancelable() 函数设置使能或者失能，默认值为 PTHREAD_CANCEL_ENABLE，表示线程可以取消。如果设置为 disable，则对该线程的取消信号会滞留在队列中，直到该线程被激活。
  2. cancelstate
    当 cancelablity 为 enable 时，cancelstate 属性决定线程的取消形式，有两种形式态，PTHREAD_CANCEL_ASYNCHRONOUS 和 PTHREAD_CANCEL_DEFERRED。前者一般是立即生效，但是不保证，后者表示线程会延迟取消，直到线程执行到取消点（cancelation point）时才生效。取消点，需要查看 ptread 的手册，是一系列系统函数，属于用户区和内核区之间的系统调用相关函数。

读写锁

前面的互斥锁会导致多个线程读数据时也会阻塞，会导致大量数据进行处理的时候，效率降低。

读写锁可以解决这个问题，读写锁分为读锁和写锁，读锁可以同时被多个线程拥有，写锁只能被一个线程拥有。

  • 如果有其它线程读数据，则允许其它线程执行读操作，但不允许写操作。
  • 如果有其它线程写数据，则其它线程都不允许读、写操作。
  • 写是独占的，写的优先级高
