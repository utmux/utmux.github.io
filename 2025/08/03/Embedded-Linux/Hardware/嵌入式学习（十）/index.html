1. 核心架构与数据结构

1.1 层次结构

GPIO 子系统分为三个主要层次：

  1. 用户接口层：

    • 提供给驱动程序使用的 API
    • Sysfs 用户接口

  2. GPIOLIB 核心层：

    • 提供统一的 GPIO 管理框架
    • 实现 GPIO 资源的分配和映射

  3. 硬件驱动层：

    • 特定 GPIO 控制器的实现
    • 通过实现 gpio_chip 接口与硬件交互

1.2 关键结构体

1.2.1 struct gpio_chip

重要成员说明：

  • label：控制器名称，用于标识
  • direction_input/output：设置 GPIO 方向
  • get/set：读取/设置 GPIO 值
  • base：该控制器 GPIO 的起始编号
  • ngpio：该控制器管理的 GPIO 数量

1.2.2 struct gpio_desc

关键作用：

  • 代表一个具体的 GPIO 引脚
  • 包含指向所属 gpio_device 的指针
  • 存储引脚的状态标志

1.2.3 struct gpio_device

主要作用：

  • 管理一个 GPIO 控制器实例
  • 连接到具体的 gpio_chip
  • 维护该控制器下的所有 gpio_desc

4. 与 Pinctrl 子系统交互

4.1 交互流程

sequenceDiagram
    participant App as 应用程序
    participant GPIOLIB as GPIOLIB
    participant Pinctrl as Pinctrl
    participant HW as 硬件
  
    App->>GPIOLIB: gpiod_get()
    GPIOLIB->>Pinctrl: 请求引脚复用
    Pinctrl->>HW: 配置引脚为GPIO
    GPIOLIB-->>App: 返回gpio_desc
    App->>GPIOLIB: gpiod_direction_output()
    GPIOLIB->>HW: 设置方向
